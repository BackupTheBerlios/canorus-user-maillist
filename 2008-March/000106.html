The correct list is the NoteEdit user mailing list. NoteEdit is no longer
actively developed but it still has many feature Canorus is still missing.
I'd be happy if Canorus would get those features this year but that mainly
depends on development activity (being unusually low this month unfortunately).

> Basically, I'd like to run noteedit on the default distro  that's
> included with my EeePC. Unfortunately noteedit isn't in the repos and
> neither are the dev tools I'd need to compile. So, I thought I would
> create a static binary with
> 
> CFLAGS=-static ./configure && make

Alternatively you could try to either recompile the debian package yourself
(being the better way instead of compiling it manually as it integrates nicely
with the system and you get rid of installation problems and missing files
like in this case.

> On my Ubuntu system and then just drop it in on Eee PC. Well, I think
> the build went okay...the binary launches runs from the "noteedit"
> directory just fine on Ubuntu. So I tarred it up, threw it on a usb
> stick, and untarred it on the Eee. When I try to run the noteedit
> binary, it complains about not being able to find libnoteedit.so.0.
> Any ideas?

Yes, this lib needs to be installed correctly in /usr/(local)/lib for example.
Also you might need to install the tse3 lib and some kde libs if there are
problems running NoteEdit. That's why I recommand to base your manual build
on the debian package.
