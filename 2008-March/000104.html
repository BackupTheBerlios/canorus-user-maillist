Nathan,

the Canarus mailing list normally doesnt handle NoteEdit issues. Further you have not a NoteEdit but a "compiling and distributing 
with Linux" problem. The best place to ask would be a Linux news group.

I dont know wich binaries are generated during NoteEdit build. With your error message I guess that a share object library 
libnoteedit.so (SO) is generated. The version is encoded in the trailing numbers. You will need to find the SO and copy it to the 
Eee in the same directory. In most cases you will find the SO.0 as a to symbolic link to the actual binary, i.e. SO.1.2.3. Copy all 
of them. The actual required libraries can be detected with "ldd NoteEdit". Probably you will need to "register" the SO with the ldd 
command. You will get help with "man ldd" and asking the Linux news group(s).
