Hi Mike,

Canorus is not yet mature as NoteEdit at the moment (but we plan to
release final version until end of 2008 which should be at least capable
as NoteEdit).

canorus-user at lists.berlios.de pravi:
> I've been using NoteEdit for a few years now to edit
> notes for export to .ly files but I just installed Canorus
> as it seems to have a future.
>
> After a problem or two (having to add 'util' to the
> TARGET_LINK_LIBRARIES line in CMakeLists.txt)
> I got it to compile and run.
>
> However, nothing seems to show up in  the view window
> when I try to add music elements to a score. Looking at
> the  source files [under View'>Source view], I see stuff
> is being added, but it doesn't show up on screen, and there
> doesn't seem to be any obvious errors showing up in the
> terminal window/command line from which I've run
> Canorus.
>   
Try adding a new sheet by Insert->New Sheet.
Otherwise, you should compile Canorus and have Python and SWIG
development files installed. This will enable Canorus scripting support
and a predefined sheet with two staffs will be added automatically when
you hit New document.
> I'm running Linux (CentOs, a variant of Redhat Enterprise),
> Qt installed fine, and I've copied the src/font files over to my TTF
> directory and so on.
>
> Any ideas? And also, is there anywhere that has some kind of
> 'Getting Started' guide to help set things up? I'm approaching stuff
> with a fair bit if NoteEdit usage, but I can't figure out any obvious
> problems.
>   
There is a small wiki page called Users guide which will eventually grow
into a whole book. But for now, it only contains some Canorus specific
behaviour and usage which is not obvious.
