Hi David.

David Cuny pravi:
> How MIDI aware is Canorus, and how much do you plan to add to it?
>
> I see that CAVoice has access to the MIDI channel and program. However, it appears that this setting is global for the voice. Does this mean that if I want to switch from bowed violins to pizzicato, I'd have to create another voice on the same staff?
>   
You can add instrument change in the middle of the voice (the keyboard
icon on the Insert toolbar on the left).
> I also don't see any way in CANote to set or adjust the velocity/volume of the instrument. This apparently means that all the voices play back at the same MIDI velocity? I assume that classes like CAFermata and CACrescendo are purely decorational?
>   
Yes, CAFermata and CACrescendo are not playable yet. But you can insert
dynamic change (p, pp, mf, ff etc. icons in the Insert toolbar). These
directly set the volume controller.
> There also appears to be no way to access MIDI controllers via events.
>   
Can you be more specific on this? For the midi playback, see CAPlayback
and CAMidiDevice classes. You can directly send the event by calling
CAMidiDevice::send(message, offset). Midi-in event is triggered on
CAMidiDevice::midiInEvent(), but is no implemented for Swig (trivial to
add though, I'll look at it).
