> CANote has diatonicPitch() method which returns a pair noteName+accs.

Thanks!

I see in the image here:

	https://canorus.berlios.de/wiki/images/8/8f/Canorus-0.6rc-bloat.png

an example of what I'm looking for. Might you consider something other than noteName() as the name of the function? Perhaps getMidiPitch() ?

> You can also call getChord() on that note or globally to gather all
> notes playing together at that time.

Does getChord get all the notes that are currently active, or the notes that were activated at that point in time? It might be less confusing if the routines were named getActiveNotes() for notes that are currently playing, and getNotesStarting() for pitches that share the same start time.

> Chords and frets aren't implemented yet, sorry. It's on our todo list.

I can probably make do with the text element. I thought that perhaps CAFunctionMarking would be available for generic markup, but it appears to be very specific in terms of what it provides. Before I can "thicken" a given line, I need two bits of information: the harmony for that note (determined by the chord), and the note function (determined by how the note approaches the next note). I could easily use arrays to hold the information internally, but there doesn't appear to be a way to display this in a generic way to the user.

Are there any example scripts available? I couldn't find any other examples of Python scripting except for what's in the Wiki webpage:

	https://canorus.berlios.de/wiki/index.php/Plugins

The description of the script is:

	"Let's create a new sheet with a staff and one voice in it. File test.py (using the descriptor above) could look like this:"

but it's essentially the same as the only script included with Canorus, and (unlike what the WIki documentation says) it creates 2 staffs, not one. It would be nice to have a more complete example, such as something that adds notes of various durations, and chords.
