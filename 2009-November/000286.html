> usually works is to install Lilypond from maintainers website and use
> their own paths which don't collide with the system fonts (lily and its

What will the future bring ? One day Lilypond >= 2.13 will be pushed from 
devel to stable. Will canorus be using the new emmentaler font in the future ?

> The second solution is
> probably that your canorus package actually doesn't contain
> Emmentaler-14.ttf file at all. Default search path for Canorus fonts is
> /usr/share/canorus/fonts, so only if emmentaler is not there, it tries

I never quite understood how fonts are managed in Linux/Fedora anyway.
The Fedora canorus package uses symlinks from 
the fonts in /usr/share/fonts/lilypond/ --> /usr/share/canorus/fonts.
If I remove the symlinks in /usr/share/canorus/fonts and replace them with 
the TTF fonts from the canorus sources that still does not work. It looks 
like even then the fonts from lilypond 2.13 are used and not the ones from 
/usr/share/canorus/fonts. Maybe some other commands are needed to install 
the fonts in the system ?
