Hi,

I am trying the lilypond-preview feature. Using lilypond 2.10.33 I 
bumped into the following problem, blocking the preview:

-----QUOTE-----

PreviewCtl: Preview triggered via main window
Started lilypond with parameters -o/tmp/qt_temp.TJ2794 /tmp/qt_temp.TJ2794
Verwerken van `/tmp/qt_temp.TJ2794'


Ontleden...


Vertolken van muziek...
[1]


Voorbewerken van grafische objecten...


Opmaakuitvoer naar `/tmp/qt_temp.TJ2794.ps'...


Converteren naar `qt_temp.TJ2794.pdf'...


PreviewCtl: Could not remove old preview file /home/m.tarenskeen/preview.pdf, error No such file or directory
Showing PDF file file:/home/m.tarenskeen/preview.pdf
PreviewCtl: Could not copy temporary file /tmp/qt_temp.TJ2794.pdf, error Cannot open /tmp/qt_temp.TJ2794.pdf for input

---- UNQUOTE ----

I can work around this problem by, from my terminal window, cd to /tmp 
and then start canorus from there. A bit awkward.
To me it looks like a bug in Lilypond. Using -o/tmp/qt_temp.xxxx causes 
lilypond to put the produces ps and midi file correctly into /tmp/ but 
the conversion of ps to pdf leads to a pdf file in my CURRENT directory 
instead of in /tmp. And canorus cannot find it because it expects to 
find it in /tmp.

questions: 

1. Is this a known bug? Has it been fixed in development 
version or reported to the Lilypond developers ? 

2. Would it be possible to let canorus use lilypond's "--ps" instead 
of the default "--pdf" ? This would speed up the process ( ps->pdf not 
needed anymore) for previewing with evince or gv. I guess you want to 
make Windows users happy who don't like Postscript ? 

BTW: In the old Noteedit I used abcm2ps for previewing. Not as powerfull 
but MUCH faster than lilypond ...
